What you receive (and why the format matters)

Most audit tools default to a static PDF export or an in-app dashboard. The first reads as commodity; the second locks the work behind a login the implementation team does not have. We ship multiple artifacts side by side, each formatted for the team that will actually use it. Pulse ships two artifacts: the PDF report and the Schema Pack. SXO Audit and Enterprise ship all three (PDF, Playbook, Schema Pack), scaled to the tier's scope.

A · The SXO Report PDF

For the executive desk

A premium, printable narrative report. Executive summary, citation state, competitive position, themed findings, prioritized roadmap, schema pack coverage, implementation handoff. Designed to read as a McKinsey-grade deliverable, not a dashboard export. The artifact you forward to the CEO.

B · The Playbook

For the implementation team

An interactive web hub at a private URL: 7 tabs covering overview, findings, schema, priority backlog, prompts, library of best practices, and FAQ. Copy-paste ready. Brand-switcher when you order Enterprise. Lives 12 months from delivery, no login required, deep-linkable so you can forward a specific section to a specific owner.

C · The Schema Pack

For the engineer

A zipped folder of paste-ready JSON-LD (JavaScript Object Notation for Linked Data) blocks, one per audited URL, pre-validated against Schema.org and Google Rich Results Test. Drop into your CMS or Next.js metadata; ship in your next PR.

Format is a pricing lever most generative-engine and AEO tools leave on the table. A static PDF reads as commodity work; a structured, interactive, personalized deliverable reads as premium. The multiple artifacts together let the same audit serve different audiences without forcing any of them to translate.
