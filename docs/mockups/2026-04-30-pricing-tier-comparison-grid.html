Pricing tier comparison grid: 3 design variants

Three design directions for the /pricing.html tier comparison grid. Today's page renders three independent cards (Pulse / SXO Audit / Enterprise) with bullet lists per card; there is no side-by-side feature-by-feature comparison. These mockups all use canonical brutalist tokens (ink #000000, paper #f4f3ef, accent #cc0000, Helvetica) scoped to .wc-scope so the framing chrome on this page does not bleed into the design surface. Open this file in a browser: file:///Users/craigkokesh/web-cited/docs/mockups/2026-04-30-pricing-tier-comparison-grid.html.

H7 lockdown applies to the PDF, Playbook, and Portfolio Summary canonicals. The pricing page is NOT one of the locked surfaces, so design changes here are in scope for review. None of the three variants below have shipped to production; they exist for morning operator review only.

Variant A Strongest scan Pure feature matrix

Replaces the existing 3-card layout with a single feature-by-feature table. Buyers can scan tier diffs row by row. Section bands group rows by purpose (audit scope, deliverables, engagement). Featured tier (SXO Audit, "Most teams pick this") gets a red accent rule above its column header and a faint tint on its cells. Tier prices live in column heads. Single CTA row at the bottom. Best fit if the priority is decision speed over storytelling.

Variant B Stepped accumulator

Three-column card layout where each higher tier visibly inherits the prior tier and adds new line items on top. SXO Audit reads "Everything in Pulse, plus..." and Enterprise reads "Everything in SXO Audit, plus..." Reduces line repetition and emphasizes the additive scope model. Featured (SXO Audit) is rendered inverted (ink background) to draw the eye. Best fit if the priority is to narrate the upgrade story; weaker for at-a-glance numeric comparison.

Variant C Cards plus by-the-numbers scoreband

Keeps the existing 3-card narrative layout (lowest delta from production) and layers on a compact "By the numbers" scoreband below: 5 numeric rows that pull the most-asked tier-diff data into one strip. A separate "Deliverables" block underneath calls out Audit Report PDF + Schema Pack (all tiers) + Playbook (SXO Audit and Enterprise only) so the upgrade motivator reads at a glance. Best fit if the priority is to keep the current page's emotional arc intact while still answering the "but how do they actually compare?" question.

Recommendation

All three variants are honest renderings of the same tier-difference matrix in the canonical brutalist system.

  • Variant A (pure feature matrix). Strongest for buyer decision speed. Replaces the cards entirely and answers "what's actually different across tiers" in one scan. Cost: loses the per-tier narrative blurb. The featured (SXO Audit) accent rule + tinted column does most of the lifting.
  • Variant B (stepped accumulator). Strongest for upgrade narrative. Reads as a story; the "Everything in Pulse, plus..." pattern reinforces why the next tier is worth more. Cost: harder to compare numerics across tiers (you have to re-read each list).
  • Variant C (cards + scoreband hybrid). Lowest production-delta. Keeps the existing 3-card narrative + layers on a 5-row numeric strip and a deliverables block. Cost: the scoreband is short by design, so it cannot show the SEO/AEO/GEO premium-scope deltas; those still live inside the cards' bullet lists.

My read: Variant A is the strongest choice if the goal is converting a cold visitor to "Start an Audit" without forcing them to read three card bullets in parallel. Variant C is the safest choice if the bigger risk is changing too much at once. Variant B is the right choice if outbound emails and content marketing are about to point at this page and you want it to read as a story for skim-readers. None of the three have shipped; the call is yours.
