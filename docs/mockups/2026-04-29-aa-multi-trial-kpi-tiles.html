How this mockup is built. With N=3 trials per (engine, prompt), n=120 total samples for Pulse / Claimzilla. CI numbers below are illustrative (computed via Wilson 95 percent on n=120 with 48 successes for "you cited 40 percent"; the proposed values would land in production from the actual pipeline output).

Surface 3: Portfolio Summary cover (Enterprise multi-brand)

The Portfolio Summary template inherits the per-brand PDF design via shared_styles (same .kpi, .kpi-cell, .kpi-label, .kpi-value, .kpi-delta classes; same Arial Black display font; same 1.5px ink rules; same --font-display 64px headline). Whatever variant is picked for Surface 2 (Audit Report PDF) automatically applies to the Portfolio Summary cover. No separate variant needed.

Recommendation

Variant B across both surfaces (Playbook + PDF + Portfolio inherits PDF). Rationale:

  • Tile typography stays byte-identical to canonical (.kpi-value still reads 40% in Arial Black 900). Visual hierarchy unchanged.
  • CI sits in slate-700 below the percent. Matches the canonical kpi-delta color so the eye reads the new line as quiet metadata, not a new highlight.
  • Variant A inline 40% +/- 9% at 22px is technically valid but pushes the visual weight outside the canonical hierarchy (the +/- competes with the percent).
  • Variant C never touches the tile, but loses the at-a-glance "we are 95 percent confident in 40 percent" read.

Variant B requires two new utility classes (.kpi-ci-below on each surface) using existing tokens (--slate-700, existing fonts, no new colors). No layout change to the canonical KPI strip; CI just inserts between .kpi-value and .kpi-delta.

Operator decision questions

  1. Pick a variant: A (inline) / B (recommended) / C (kpi-delta-only).
  2. Confirm "+/- 9%" wording. Alternatives: "+/- 9 pts" (clearer that the unit is percentage points), "[31% to 49%]" range form, "9% margin" phrase.
  3. For multi-brand Enterprise audits, the Portfolio Summary cover automatically inherits whatever variant is picked for the per-brand PDF. Confirm that is intended (vs Portfolio cover staying point-estimate while per-brand PDFs show CI).
  4. Apply CI to Section 3 Competitive Position table (per-competitor share entries) too? Defaults to YES; if NO, the table stays point-estimate while Section 1 shows CI.
